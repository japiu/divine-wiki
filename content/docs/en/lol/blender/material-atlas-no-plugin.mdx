Then let's continue.

First things first, what is a material atlas? Well, ideally we want to have the 3D model textures packed into a single image so we can import them into league and sometimes when you download a model, its material view looks like this:

<figure>
  <img
    id="img-1"
    src="/wiki-images/j3-img-1.png"
    alt="List of materials of an imported 3D model"
  />
  <figcaption>Image 1. List of materials of an imported 3D model</figcaption>
</figure>

As you can see there’s a long list of materials and each one is tied to an image containing a portion of your model’s textures. We will narrow it down to just 1.

To do so we need to follow these steps:

1. UV unwrap your model into a new blank image **using a new uv map**.
2. Setup the baking
3. Bake the textures into our blank image.
4. Final touch and cleanup

# Step 1

<Callout>
This is probably the hardest part of the process so once finished you will be 80% done.

For the sake of this tutorial we will skip the part of “good UVs” and we will let blender do it all automaticaly for us, but bear in mind that it will always be better if you manually create seams for your model and unwrap them properly.

</Callout>

# Step 2

The baking tells blender to transfer certain parts of how the 3D model is looking now into another place, usually a texture image.

For this step we’re going to use the <b>shading</b> tab from the tab menuat the top of the blender window.

In the left side, make sure to open our previously cretaed “one_texture” image which should be black yet.

<figure>
  <img id="img-15" src="/wiki-images/j3-img-15.png" alt="Shading tab" />
  <figcaption>Image 15. Shading tab</figcaption>
</figure>

Now comes the boring part. go to the right side panel, the material view, and select the first one. Then, in the node panel (the boxes right below our 3D model) press F3 and type “Image texture”

<figure>
  <img
    id="img-16"
    src="/wiki-images/j3-img-16.png"
    alt="Creating a new image texture"
  />
  <figcaption>Image 16. Creating a new image texture</figcaption>
</figure>
Now select our “one_texture” image from the “picture” icon. You should have
somethign like this
<figure>
  <img
    id="img-17"
    src="/wiki-images/j3-img-17.png"
    alt="Creating a new image texture output"
  />
  <figcaption>Image 17. Creating a new image texture output</figcaption>
</figure>
By the way, I like to change the color space to sRGB, the rest leave it as is.

Notice how you have a white highlight around the border of the box? That means it is selected, <b>this is also important for the baking</b>.

Once done, hit ctrl+c to copy the node, and go through each one of the remaining materials in the right side panel list and hit ctrl+v to paste it.

If you did it correctly, you will now have one box like the <a href="#img-17">Image 17</a> per material, make sure they are all selected.

# Step 3

Go to the render tab in the right side panel

<figure>
  <img id="img-18" src="/wiki-images/j3-img-18.png" alt="Render tab" />
  <figcaption>Image 18. Creating a new image texture output</figcaption>
</figure>
<Callout type="warning">
  Change the render engine from EEVEE to Cycles since EEVEE doesn’t support
  baking.
</Callout>

Enter the “Bake” submenu there and select these settings:

<figure>
  <img
    id="img-19"
    src="/wiki-images/j3-img-19.png"
    alt="Bake submenu settings"
  />
  <figcaption>Image 19. Bake submenu settings</figcaption>
</figure>
A diffuse map bakes the color of the texture, we disable direct and indirect
since we dont want any lighting interfering.

<b>Before hitting “Bake” button, make sure of this</b>

1. The selected UV map is the one you created new.
2. There’s a new selected image texture node for each material.
3. Your 3D model is also selected

<figure>
  <img id="img-20" src="/wiki-images/j3-img-20.png" alt="Check bake settings" />
  <figcaption>Image 20. Check bake settings</figcaption>
</figure>

Once ready, hit bake.

# Step 4

Time to clean up and test the result.

Go back to layout view (at the top menu bar), your 3D model should look good by now, let’s mess it up a bit. First, let’s create a new material to hold our new texture.

<figure>
  <img id="img-23" src="/wiki-images/j3-img-23.png" alt="Create new material" />
  <figcaption>Image 23. Create new material</figcaption>
</figure>
Give it a name, like “one_texture” and switch to the shading layout then put the
nodes like in the following image.
<figure>
  <img id="img-24" src="/wiki-images/j3-img-24.png" alt="Setting nodes" />
  <figcaption>Image 24. Setting nodes</figcaption>
</figure>
Go back to the layout tab and delete all materials except the one created by us.
You will end up with a mess like this

  <img id="img-25" src="/wiki-images/j3-img-25.png" alt="Material doomsday" />

camera icon so that your new uv map is the one having it active and see the
magic.

  Now the final touch, remove the old “UVMap” and rename the new one to the same
  “UVMap” name. I’m not sure but I read somewhere that the UVMap name is
  important for the skins to work in league.
</Callout>
