# Understanding .bin, .dds, and .tex Files

Modding League of Legends involves working with several key file types. Master these three formats and you'll be able to edit anything in the game.

---

##  Quick Reference

| File | What It Is | Convert With |
|------|-----------|--------------|
| `.bin` | Logic & config data | [RitoBin](/docs/tools/Ritobin) |
| `.dds` | Standard texture format | Direct edit |
| `.tex` | Riot's texture format | [tex2dds](/docs/tools/tex2dds) |

## .bin Files – Data & Logic

`.bin` files store **all the logic** in League — particle systems, skin definitions, animations, sounds, and more.

| Contains | Examples |
|----------|----------|
| Particle data | VFX colors, sizes, timing |
| Skin definitions | Model paths, texture paths |
| Animation configs | Animation names, blending |
| Sound configs | Audio triggers, volumes |

### Workflow

1. Extract `.bin` using [Obsidian](/docs/tools/obsidian)
2. Convert to `.py` using [RitoBin](/docs/tools/Ritobin)
3. Edit in VS Code or Notepad++
4. Convert back to `.bin`
5. Add to your mod folder

> 💡 `.py` files are human-readable — you can search, edit, and understand them easily.

</Tab>

## .dds Files – Textures

`.dds` (DirectDraw Surface) is the **standard texture format** used for:

| Usage | Examples |
|-------|----------|
| Champion models | Skin textures, faces |
| Particles | VFX sprites, glows |
| UI/HUD | Icons, ability panels |

### Editing Tools

[Photoshop | Best option with NVIDIA DDS plugin](https://www.adobe.com/products/photoshop.html)
[GIMP | Free alternative with DDS support](https://www.gimp.org/)
[Paint.NET | Lightweight with DDS plugin](https://www.getpaint.net/)

### Export Settings

| Setting | Value |
|---------|-------|
| Format | `.dds` |
| Compression | **BC3 / DXT5** |
| Mipmaps | ✅ Enabled |

> ⚠️ Wrong compression = broken textures in-game!

</Tab>

## .tex Files – Riot's Format

`.tex` is Riot's **proprietary texture format** found in `.wad.client` files.

| Status | Info |
|--------|------|
| ❌ Cannot edit directly | It's a compiled format |
| ✅ Must convert first | Use tex2dds |
| ✅ Convert back after | Before reimporting |



[tex2dds | Convert .tex ↔ .dds files](/docs/tools/tex2dds)

> 💡 Always keep the original `.tex` as backup before converting.

##  Complete Workflow Chart

| Task | File | Tool | Output |
|------|------|------|--------|
| Edit VFX/particles | `.bin` | RitoBin → VS Code | `.bin` |
| Edit champion skin | `.dds` | Photoshop/GIMP | `.dds` |
| Extract game texture | `.tex` | tex2dds | `.dds` |
| Reimport texture | `.dds` | tex2dds | `.tex` |
| Browse game files | `.wad` | Obsidian | Extracted files |
